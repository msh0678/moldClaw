# 설정 UX 개선 제안서

> **주의**: 이 문서는 제안사항만 포함합니다. 코드 수정은 sanghyuck의 검토 후 진행.

---

## 현재 문제점

### 1. Gmail 중복 존재
- **일반 설정 > Gmail**: 환경변수 기반 간단 연동
- **고급 설정 > Gmail Settings**: 상세 OAuth 설정
- 사용자 혼란 유발

### 2. '일반' 설정의 정체성 모호
- 현재: "설정 현황 보기" 역할
- 문제: AI 모델/메신저는 별도 탭으로 충분
- 불필요한 depth 추가

### 3. Tools/Skills 접근성
- 현재: '고급' 탭에만 존재
- 자주 쓰는 도구 (Brave Search 등)는 쉽게 접근해야 함

---

## 개선 제안

### Option A: '일반' 탭 제거
```
설정 탭 구조:
├── AI 모델 (현재 그대로)
├── 메신저 (현재 그대로)
├── 도구 (현재 '고급 > 도구')
├── 스킬 (현재 '고급 > 스킬')
├── TTS (현재 '고급 > TTS')
└── Gmail (통합 - OAuth 포함)
```

**장점:**
- 불필요한 중간 depth 제거
- Gmail 중복 해결
- 모든 설정이 동일 레벨

**단점:**
- 초보자에게 탭이 많아 보일 수 있음

---

### Option B: '일반' 탭을 '필수 설정'으로 리브랜딩
```
설정 탭 구조:
├── 필수 설정 (AI 모델 + 메신저 + 추천 도구)
│   ├── AI 모델 선택
│   ├── 메신저 연결
│   └── 추천 도구 (Brave Search, Gmail 등)
│
└── 고급 설정 (나머지)
    ├── 전체 도구 목록
    ├── 전체 스킬 목록
    ├── TTS
    └── 기타
```

**장점:**
- 온보딩 친화적
- 필수 vs 선택 구분 명확

**단점:**
- 도구가 두 곳에 존재 (추천 + 전체)

---

### Option C: 카테고리 기반 (권장)
```
설정 탭 구조:
├── 연결 (메신저 + Gmail)
├── AI (모델 + TTS)
├── 확장 (도구 + 스킬)
└── 시스템 (경로, 포트 등)
```

**장점:**
- 논리적 그룹핑
- 중복 제거
- 확장 가능

---

## Gmail 통합 제안

현재 두 가지 Gmail 설정을 하나로 통합:

```tsx
// 통합된 Gmail 설정 컴포넌트
<GmailIntegration>
  <Step1>Google Cloud Console 설정</Step1>
  <Step2>OAuth 자격 증명</Step2>
  <Step3>토큰 입력</Step3>
  <Step4>연결 테스트</Step4>
</GmailIntegration>
```

위치: **연결** 카테고리 또는 **스킬** 내 Google Workspace
